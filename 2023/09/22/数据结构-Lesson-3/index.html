  • 数据结构

  我们认识三种新的线性数据结构，它们是 栈、队列 和 优先级队列。

  与线性表不同的是，它们都是限制存取位置的线性结构

§ 1 栈 Stack

1.1 栈的定义

  • 只允许在一端插入和删除的线性表
  • 允许插入和删除的一端称为栈顶（top）、另一端称为栈底（bottom）
  • 特点：后进先出（LIFO）

1.2 栈的抽象数据类型

存储栈元素的容器也可以使用数组。用数组表示的栈也称为顺序栈

  • 栈溢出的处理函数，通常列为private
  • 入栈函数
  • 出栈函数
  • 取栈顶元素

顺序栈的溢出

  • 上溢（Overflow）

      当栈中已经有maxsize个元素时，再做进栈运算

  • 下溢（Underflow）

      对空栈进行出栈运算

如何合理进行栈空间分配，以避免栈溢出或空间的浪费？

  • 双栈共享一个栈空间

    • 两个栈共享一个数组空间V[maxSize]

    • 设立栈顶指针数组t[2]和栈底指针数组b[2]，t[i]和b[i]分别指示第i个栈底栈顶与栈底

    • 初始t[0]=b[0]=-1,t[1]=b[1]=maxSize

    • 栈满条件：t[0]+1==t[1] //栈顶指针相遇

    • 栈空条件：t[0]==b[0]或t[1]==b[1]

    • push函数

    • Pop函数

  • 栈顶链接存储方式–链式栈

    • 链式栈无栈满问题，空间可扩充

    • 插入与删除只在栈顶处执行

    • 链式栈的栈顶在链头

    • 适合于多栈操作

栈的应用：表达式的计算

  • 算术表达式有三种表示：

    • 中缀（infix）表示

      <操作数> <操作符> <操作数>,如 A+B;

    • 前缀（prefix）表示

      <操作符> <操作数> <操作数>,如 +AB;

    • 后缀（postifx）表示

      <操作数> <操作数> <操作符>,如AB+;

后缀表达式

  后缀表达式，又称逆波兰式，是一种不包含括号，运算符放在两个运算对象的后面，所有的计算按运算符出现的顺序，严格从左向右进行（不再考虑运算符的优先规则）。例如，中缀表达式9 + (3 - 1) * 3 + 10 / 2可以转换为后缀表达式9 3 1 - 3 * + 10 2 / +。

§ 2 栈与递归

  • 递归的定义

    若一个对象部分地包含它自己，或用它自己给自己定义，则称这个对象是递归的；若一个过程直接地或间接地调用自己，则称这个过程是递归的过程。

  • 有以下三种情况常常需要用到递归方法

    • 定义是递归的

      例如，阶乘函数

      n!= { 1（当n=0时） n*（n-1）！（当n≥1时）}

      求解阶乘函数的递归算法

      long Factorial(long n){

      ​ else return n*Factorial(n-1);

      }

    • 数据结构是递归的

      单链表结构，如果它的指针域是nullptr或者单链表，都可以是一个单链表

    • 问题的解法是递归的

      • 搜索链表最后一个结点并输出其数值

        递归找链尾

        template<class E>

        void Print(ListNode<E> * f){

        ​ if(f->link==nullptr){

        ​ cout<<f-><<endl;

        ​ }

        ​ Print(f->link);

      • 汉诺塔问题的解法

        • 如果n=1，则将这个盘子直接从A柱移动到C柱上。

        • 否则，执行以下三步

          1>. 用C柱做过渡，将A柱上的（n-1）个盘子移动到B柱上

          2>. 将A柱上最后一个盘子直接移到C柱上

          3>.用A柱做过渡，将B柱上的（n-1）个盘子移到C柱上

  • 什么时候运用递归？

    • 子问题应与原问题做同样的事情，且更为简单；
    • 把一个规模比较大的问题分解为一个或若干规模比较小的问题，分别对这些比较小的问题求解，再综合它们的结果，从而得到原问题的解。

    分而治之策略（分治法）

  • 使用递归的前提

    比较小的问题的求解方法与原来问题的求解方法一样，并且难度（或规模）更小。

  • 构成递归的条件

    不能无限制的调用自身，而必须有一个出口，在化简为非递归状况时可以直接处理

  • 递归过程与递归工作栈

    • 递归过程在实现时，需要自己调用自己

    • 层层向下递归，退出时的次序正好相反

    • 主程序第一次调用递归过程是外部调用

    • 递归过程每次递归调用自己为内部调用

    • 它们返回调用它的过程的地址不同

        这里是指在内存中的位置不同。每次调用一个函数时，系统会为该函数分配一块儿栈空间，以存储该函数的参数，局部变量和返回地址等信息。当函数返回时，该栈空间会被释放。因此，每调用一个函数时，它的地址都可能不同。

        递归的外部调用地址是指第一次调用递归函数是的栈空间的地址，它通常位于主函数或其它函数的栈空间之上。

        递归的内部调用的地址是指每次在递归函数中调用自己时的栈空间地址，它通常位于上一次递归函数的栈空间之上。

  • 递归过程 与 非递归过程

    • 递归过程简洁，易编，易懂
    • 但有的时候递归过程效率低，重复计算多
    • 改为非递归过程的目的是提高效率

  • 递归改非递归的方法

    • 通过迭代实现非递归过程（高效）

    • 借助栈实现非递归过程（功能等价）

§ 3 队列 Queue

1.定义

  • 队列是只允许在一端删除，在另一端插入的顺序表，也就是先进先出
  • 允许删除掉一端叫做队头（front），允许插入的一端叫做队尾（rear）

2.特性

  • 先进先出（FIFO，First In First Out）

  • 队列的抽象数据类型

    • 进队：rear = rear +1，新元素在rear处加入
    • 出队：front = front +1，取出下标为front的元素
    • 队空时：rear == front
    • 队满时：rear == maxSize -1

      解决假溢出的办法之一：将队列元素存放数组首尾相接，形成循环（环形）队列

  • 循环队列（Circular Queue）

    • 队列存放数组被当作首尾相接的表处理

    • 队头、队尾指针加1时从maxSize-1直接进到0，可以用取模运算实现

    • 队头指针进1：front=（front+1）%maxSize;

    • 队尾指针进1：rear=（rear+1）%maxSize;

    • 队列初始化：front=rear=0；

    • 队空条件：front==rear；

    • 队满条件：（rear+1）%maxSize==front;

    • 循环队列操作的定义

  • 链式队列-队列的链接表示

    • 队头在链头，队尾在链尾
    • 链式队列在进队时无队满问题，但有队空问题
    • 队空条件为 front == nullptr

    链式队列类的定义

  • 队列的应用

    逐行打印二项式 (a+b)^i的系数（杨辉三角）

§ 4 优先级队列 Priority Queue

  • 定义：优先级队列是不同于先进先出队列的另一种队列。每次从队列中取出的是具有最高优先权的元素

    • 优先权是根据具体问题而定的
    • 出现相同优先级的元素时，按FIFO的方式处理
  • template <class E> 
void PQueue<E>::adjust() {
     E temp = pqelements[count-1];
     //将最后元素暂存再从后向前找插入位置
     for (int j = count-2;  j >= 0;  j--)
         if (pqelements[j] <= temp) break; 
         else pqelements[j+1] = pqelements[j];
     pqelements[j+1] = temp;
}
